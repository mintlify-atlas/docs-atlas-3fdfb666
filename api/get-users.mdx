## Endpoint

<ParamField path="method" type="string" default="GET">
  GET
</ParamField>

<ParamField path="path" type="string">
  `/users`
</ParamField>

## Overview

This endpoint returns the complete demo dataset including all users, resources, and actions available in the application. It's primarily used by the frontend to populate the UI with available options.

This is a simple, read-only endpoint that returns static demo data. No request parameters are required.

## Request

No request body or parameters required.

### Example Request

```bash
curl -X GET https://[your-api-id].execute-api.[region].amazonaws.com/Prod/users
```

## Response

### Success Response (200)

<ResponseField name="users" type="array" required>
  Array of user objects with their attributes.
  
  <Expandable title="User object properties">
    <ResponseField name="id" type="string">
      Unique user identifier
    </ResponseField>
    <ResponseField name="name" type="string">
      Full name of the user
    </ResponseField>
    <ResponseField name="role" type="string">
      User's role (Analyst, Admin, Auditor)
    </ResponseField>
    <ResponseField name="department" type="string">
      Department the user belongs to
    </ResponseField>
    <ResponseField name="clearance" type="number">
      Clearance level (1-3)
    </ResponseField>
    <ResponseField name="avatar" type="string">
      Emoji avatar for UI display
    </ResponseField>
    <ResponseField name="description" type="string">
      Description of the user's permissions and role
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="resources" type="array" required>
  Array of resource/document objects.
  
  <Expandable title="Resource object properties">
    <ResponseField name="id" type="string">
      Unique resource identifier
    </ResponseField>
    <ResponseField name="label" type="string">
      Display label with emoji
    </ResponseField>
    <ResponseField name="department" type="string">
      Department that owns the resource
    </ResponseField>
    <ResponseField name="classification" type="string">
      Security classification (confidential, restricted, internal)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="actions" type="array" required>
  Array of available action strings.
</ResponseField>
